扫描结果差异化分析

对于所有客户机，可以设置不同的bash监视脚本。对于一般不大的客户机网络，可以设置简单的cronjob来执行外部端口差异化分析。

下面为一个nmap扫描脚本（需要替换端口）：

#!/bin/bash
if [ ! -e /opt/nmap_diff ]; then
    mkdir /opt/nmap_diff
fi
d=$(date -d today +%Y-%m-%d)
y=$(date -d yesterday +%Y-%m-%d)
/usr/bin/nmap -T4 -oX /opt/nmap_diff/scan_$d.xml 192.168.1.0/24 > /dev/null 2>&1
if [ -e /opt/nmap_diff/scan_$y.xml ]; then
    /usr/bin/ndiff /opt/nmap_diff/scan_$y.xml /opt/nmap_diff/scan_$d.xml > /opt/nmap_diff/diff.txt
fi

crontab -e 开始创建任务，crontab -l 显示创建的任务

0 7 * * * /usr/bin/bash /tmp/diff.sh    #每天早上7点执行diff.sh脚本

Masscan 适用于范围较大的网络环境，扫描速度快于nmap

实验：网络diff扫描器：

  • 自己构建一个端口列表
  • 实现 nmap banner
  • 保持对端口的历史追踪
  • 建立通知系统
#!/bin/sh
TARGETS="192.168.1.0/24"
OPTIONS="-v -T4 -sV"
# OPTIONS="-v -T4 -sV -p 22,80,8080,3389"
d=$(date +%Y-%m-%d)
y=$(date +%Y-%m-%d)
cd /opt/nmap_diff
nmap $OPTIONS $TARGETS -oX scan_$d.xml > /dev/null
if [ -e /opt/nmap_diff/scan_$y.xml ]; then
    ndiff scan_$y.xml scan_$d.xml > diff.txt
    curl -F "file=@diff.txt;filename=diff_$d.txt" https://slack.com/api/files.upload
fi
